# pi-posture

A live posture policy layer / on-demand harness for Pi. Switch Pi's working posture mid-session to change prompt guidance, context policy, tool and thinking overrides, runtime state, and UI status or widget display.

Built-in postures each carry a **policy** — internal runtime behavior implemented via hooks — plus declarative fields configured either in code or via JSON config.

## Built-in postures

| Posture | Policy intent | Description |
|---------|--------------|-------------|
| `default` | No-op — no posture overlay, no runtime hooks. | Normal Pi behavior. Restores previously active tools and thinking level when switching away from a non-default posture. |
| `agent` | **Autonomous progress** — pushes forward with verification and escalation guidance. | Delegated implementation posture. The policy injects a dynamic system prompt urging autonomous progress, verification after changes, and clear escalation when blocked. |
| `assist` | **Human-led pair programming** — user keeps ownership, agent suggests next steps. | Collaborative posture. The policy injects guidance that keeps the user as primary implementer while the agent inspects, verifies, and proposes narrow next moves. |
| `learn` | **Hint-first / Socratic teaching** — cognitive guidance, not permissions. | Tutor posture. The policy injects guidance for Socratic questioning, micro-exercises, and explanation-before-code. The full toolset remains available — learning mode is cognitive, not restrictive. |
| `review` | **Evidence-first critique** — read-mostly, cite sources, explain risks. | Critique posture. The policy injects guidance for evidence-backed findings and suggests fixes without modifying files unless asked. |

All built-in postures (except `default`) may add **bounded dynamic guidance** beyond their `promptOverlay` — for example, the agent policy appends `## Agent Guidance` instructions and the learn policy appends `## Learn Guidance` instructions each turn. This dynamic content is generated by the posture's internal policy hooks, not from a config file.

## Commands

```text
/posture             — interactive picker (UI) or list (headless)
/posture list        — list all registered postures
/posture status      — show active posture summary
/posture inspect     — show full active posture details including config errors
/posture <name>      — switch to a posture by name or alias
```

### Aliases

| Alias | Resolves to |
|-------|-------------|
| `off`, `reset`, `vanilla` | `default` |
| `teacher`, `tutor`, `study` | `learn` |
| `pair` | `assist` |
| `autonomous`, `execute` | `agent` |

## Custom postures

Define custom postures in `postures.json` — one of:

- `~/.pi/agent/postures.json` (global)
- `.pi/postures.json` (project-local, overrides global)

Project-local config is loaded from the `.pi` directory (matches Pi's standard project config directory). Config errors never break Pi startup; run `/posture inspect` to see any issues.

### Supported config fields

| Field | Type | Description |
|-------|------|-------------|
| `label` | `string` | Display label for the posture. |
| `description` | `string` | Short description shown in lists and the startup picker. |
| `promptOverlay` | `string` | Injected into the system prompt inside `<pi_posture>` tags when the posture is active. |
| `contextPolicy` | `object` | Filter project context files. Values: `{ global: "inherit" \| "suppress", project: "inherit" \| "suppress" }`. |
| `activeTools` | `string[]` | Override the set of active tool names. Unknown tools are silently dropped. |
| `thinking` | `"off" \| "minimal" \| "low" \| "medium" \| "high" \| "xhigh"` | Override the thinking level. |
| `interactionStyle` | `"autonomous" \| "assistive" \| "review" \| "socratic" \| "custom"` | Declarative hint about the intended interaction mode. Not a runtime hook. |
| `mutationPolicy` | `"allow" \| "guarded" \| "read-mostly"` | Declarative hint about mutation permissions. Not a runtime hook. |
| `answerPolicy` | `"direct" \| "hint-first" \| "explicit-request"` | Declarative hint about answer style. Not a runtime hook. |
| `statusLabel` | `string` | Declarative label shown in `/posture inspect`; reserved for future status policy use. |
| `dynamicPrompt` | `"none" \| "objective-aware" \| "verification-focused" \| "socratic" \| "review-focused"` | Declarative hint for future dynamic prompt behavior. Not a runtime hook. |

> **Important**: The `policy` field with executable hook functions is reserved for built-in and programmatic use. JSON config files **cannot** define `policy` entries.

### Example

```json
{
  "postures": {
    "coach": {
      "label": "Coach",
      "description": "Ask Socratic questions before giving implementation advice.",
      "promptOverlay": "Ask one concise guiding question before proposing a solution.",
      "interactionStyle": "socratic",
      "answerPolicy": "hint-first",
      "mutationPolicy": "guarded",
      "contextPolicy": {
        "global": "inherit",
        "project": "inherit"
      },
      "thinking": "medium"
    }
  },
  "aliases": {
    "socratic": "coach"
  },
  "startupPicker": {
    "enabled": true,
    "onlyWhenUnset": true,
    "include": ["default", "agent", "assist", "learn", "review"],
    "reasons": ["startup", "new", "resume", "fork"],
    "timeoutMs": 10000
  }
}
```

## Startup picker

Shown at session start when:

- `startupPicker.enabled` is `true`
- The session has a UI (`ctx.hasUI`)
- The session-start reason is in `startupPicker.reasons`
- Either `onlyWhenUnset` is `false` or no previous posture was restored from the session

Accepts `true` / `false` to quickly enable or disable with defaults, or a full object.

## Context policy

Built-in postures keep the full toolset available. Learning mode is cognitive guidance, not a permissions mode. When `contextPolicy` is set, global context paths (from `~/.pi/agent` or `~/.agents`) and project context files can be selectively suppressed.

## Project config trust note

Project-local config is loaded only when Pi marks the project trusted through `ctx.isProjectTrusted()`. Extension contexts without that trust API skip project config by default.
